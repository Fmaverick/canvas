# 功能需求优先级与开发清单

## 1. 文档目标

本文档用于承接当前新增的一批产品需求与优化诉求，统一整理为可执行的开发清单，明确：

- `P0 / P1 / P2` 优先级
- 前端改动点
- 后端改动点
- 数据库新增字段或新增表建议
- 与现有架构和数据模型的衔接方式

本文件作为现阶段产品迭代和排期拆解依据，优先用于指导工作台、画布、资源库、工作流封装和主体库扩展。

## 2. 关键决策

### 2.1 模特库实现原则

模特库不单独新建一套与主体库平行的数据体系，明确以现有主体库为基础扩展。

落地原则：

- 数据层继续复用 `library_items`
- 模特统一归入 `kind = subject`
- 使用 `entity_type = model` 或 `entity_type = person` 区分模特主体
- 模特专属信息写入 `profile_meta`
- 模特风格、定位、用途继续复用 `tags`
- 前端产品层可以单独显示为“模特库”视图，但底层仍属于主体库能力的一部分

这样可以避免后续出现：

- 主体库和模特库重复上传同一批资源
- 画布引用逻辑分叉
- 搜索、筛选、标签体系维护两套规则

### 2.2 工作流封装实现原则

当前系统已有单节点模板方向，但本批需求中的“封装工作流”明确是工作流级模板，不等同于单节点模板。

封装工作流至少需要包含：

- 节点类型组合
- 节点默认提示词
- 节点默认参数
- 节点连接关系
- 模板分类
- 模板标签
- 从模板一键创建到当前画布

因此建议新增工作流模板对象，不将其硬塞入现有 `node_templates`。

### 2.3 优先级执行原则

本阶段以“先稳定生产，再提升复用，再做体验增强”为执行顺序：

1. 先修 `P0` 稳定性和关键交互
2. 再做 `P1` 复用能力与核心资产能力
3. 最后推进 `P2` 的效率优化和体验增强

## 3. P0 开发清单

### 3.1 提示词输入稳定性修复

当前状态：已完成（代码修改完成，文件诊断检查通过）

#### 需求目标

- 修复提示词输入时内容突然消失
- 修复在段落中间输入却跳到末尾
- 修复无法回退的问题

#### 前端改动点

- 画布节点编辑区改为“本地草稿优先”
- 运行态刷新不得覆盖正在输入的草稿内容
- 输入框增加焦点态保护和脏数据判断
- 保存与运行动作分离，避免边输入边被远端状态覆盖
- 文本节点、分镜节点、图片节点、视频节点统一复查 prompt draft 同步逻辑

#### 后端改动点

- 保持节点运行态更新和节点输入保存解耦
- 若运行时状态回写接口会覆盖节点主输入，需拆分字段更新范围
- 保证节点状态刷新只更新 runtime/output，不覆盖用户当前 prompt 输入

#### 数据库建议

优先不新增主表，仅按需补充字段：

- `canvas_nodes.prompt_updated_at`
- 可选：`canvas_nodes.last_editor_session_id`

### 3.2 批量执行重构

#### 需求目标

- 批量执行可真正运行
- 自动建立对应数量的最终视频节点
- 支持两种模式：
  - 单节点批量：提示词相同
  - 多节点批量：素材和逻辑（封装工作流）相同

#### 前端改动点

- 新增批量执行配置面板
- 明确区分“单节点批量”和“多节点批量”
- 支持输入运行次数
- 展示将被复制和执行的最终节点范围
- 新增批量结果列表、批量下载和失败状态反馈

#### 后端改动点

- 重定义批量执行语义，不能仅重复调用当前节点运行
- 支持以末端视频节点为目标自动派生执行节点
- 支持按链路拓扑执行
- 支持批量执行实例和结果映射
- 支持批量执行完成后的结果聚合与下载

#### 数据库建议

在现有批量运行表基础上扩展：

- `node_run_batches.batch_mode`
- `node_run_batches.target_node_type`
- `node_run_batches.derive_strategy`
- `node_run_batches.template_snapshot_json`
- `node_runs.derived_target_node_id`
- `node_runs.source_batch_item_key`
- `node_runs.display_order`

如现有结构不足，可新增：

- `batch_run_items`

### 3.3 视频节点交互重构

当前状态：进行中（已完成 AI 输入面板左右布局，左侧参数配置、右侧当前视频预览）

#### 需求目标

- 点击节点时优先预览
- 配置面板不再无条件打断预览
- 一键下载更直达
- 放大预览和下载动作路径更短
- AI 输入面板展开时，仍可直接预览当前选中视频节点的视频结果

#### 前端改动点

- 视频节点主体点击默认进入预览态
- 配置改为独立入口，例如抽屉、标签页或显式编辑按钮
- 下载按钮前置到节点可见区域
- 预览弹层内保留直接下载
- 视频节点按结果比例展示，而不是强制统一裁切
- 在 AI 输入面板内增加当前选中视频节点的视频预览区
- 面板内视频预览与参数输入同屏展示，避免用户为了看结果反复关闭配置面板
- 面板内预览区保留播放、暂停、放大预览和下载入口
- AI 输入面板采用左右布局：左侧保持正常参数配置区，右侧固定为当前视频节点预览区

#### 后端改动点

- 统一返回视频元数据
- 若当前结果快照缺少比例、封面、下载地址，需补齐任务结果结构

#### 数据库建议

优先写入结果元数据：

- `task_results.meta.aspect_ratio`
- `task_results.meta.cover_asset_id`
- `task_results.meta.duration_ms`
- `task_results.meta.width`
- `task_results.meta.height`
- `task_results.meta.download_url_snapshot`

### 3.4 连线入口命中区优化

当前状态：已完成（代码修改完成，文件诊断检查通过）

#### 需求目标

- “入”“出”按钮在缩放后仍容易点击

#### 前端改动点

- 增大连接按钮视觉尺寸
- 扩大实际点击热区
- 缩放时设置最小可点击尺寸
- 增加 hover 高亮和可连接状态反馈

#### 后端改动点

- 无核心后端改动

#### 数据库建议

- 无新增字段和表要求

### 3.5 资源库批量导入基础版

#### 需求目标

- 资源库支持批量导入
- 降低当前复杂配置成本

#### 前端改动点

- 新增批量导入弹窗
- 支持批量上传后统一打标签
- 支持导入后统一归类到主体或场景
- 支持导入结果预览和失败提示

#### 后端改动点

- 批量申请上传凭证
- 批量完成上传回调
- 批量创建资源记录
- 批量设置封面和标签

#### 数据库建议

建议新增：

- `import_batches`

建议扩展：

- `library_items.import_batch_id`
- `assets.import_batch_id`

## 4. P1 开发清单

### 4.1 封装工作流

#### 需求目标

- 能封装预设工作流
- 放在画布左侧有单独入口
- 有专门用于建立和存放封装工作流模板的界面
- 支持按“画面效果”“内容类型”等分类

#### 前端改动点

- 左侧导航新增“封装工作流”
- 新增工作流模板列表页
- 新增工作流模板创建和编辑页
- 画布左侧资源区支持一键插入模板
- 支持模板分类筛选、搜索和标签浏览

#### 后端改动点

- 新增工作流模板 CRUD
- 新增模板分类 CRUD
- 新增从模板应用到画布能力
- 保存模板时记录节点和边快照

#### 数据库建议

建议新增表：

- `workflow_templates`
- `workflow_template_nodes`
- `workflow_template_edges`
- `workflow_template_categories`

### 4.2 模特库

#### 需求目标

- 建立自有模特数字人资产
- 避免侵权和甲方重复修改
- 模特支持标签和分类
- 面向服装类和配饰类电商场景沉淀标准化素材

#### 前端改动点

- 在主体库中新增“模特”筛选视图
- 模特卡片展示更强调封面图和标签
- 模特详情补充模特专属字段编辑区
- 模特导入和图片预览沿用主体库能力

#### 后端改动点

- 主体查询支持 `entity_type = model/person`
- 支持模特标签筛选和组合搜索
- 支持模特专属字段写入 `profile_meta`
- 支持画布节点仅筛选模特主体引用

#### 数据库建议

不新增独立模特主表，基于 `library_items` 扩展：

- `library_items.kind = subject`
- `library_items.entity_type = model | person`
- `library_items.profile_meta` 中增加：
  - `gender`
  - `age_range`
  - `makeup_style`
  - `hair_style`
  - `outfit_style`
  - `ecommerce_category`
  - `shoot_texture`
  - `usage_notes`
  - `preset_type`

标签继续复用 `tags`：

- 人设标签：甜美、御姐、辣妹
- 风格标签：Y2K、帅气、时尚、复古
- 业务标签：服装、配饰

### 4.3 资源库体验重构

#### 需求目标

- 资源库支持更直接的预览
- 支持直接添加新的主体资源
- 支持批量导入
- 支持图片直接拖入画布
- 支持打标签和自动归类

#### 前端改动点

- 资源库从“表单编辑型”改成“卡片预览型”
- 增加资源快速新建入口
- 增加图片拖入画布能力
- 拖入后支持选择“仅生成节点”或“入库并生成节点”
- 支持导入后快速打标签

#### 后端改动点

- 支持拖入图片后的临时资产处理
- 支持从画布结果反向沉淀为主体资源
- 支持标签推荐和资源归类策略

#### 数据库建议

建议扩展：

- `library_items.category_path`
- `library_items.source_type`
- `assets.source_task_id`
- `assets.source_node_id`
- `assets.asset_role`

### 4.4 提示词支持 @ 图片资产

#### 需求目标

- 提示词中可直接 @ 对应图片资产
- 被关联的图片在节点界面中可见

#### 前端改动点

- 提示词输入框增加 `@` 联想选择
- 引用资产显示为可视 token
- 节点配置面板显示已关联图片

#### 后端改动点

- 将 prompt 中的资产引用解析为结构化引用
- 运行时统一把结构化资产注入 prompt 和 reference assets

#### 数据库建议

建议扩展：

- `canvas_nodes.resource_refs`

或新增：

- `canvas_nodes.inline_asset_refs_json`

## 5. P2 开发清单

### 5.1 画布缩放与拖拽体验优化

#### 需求目标

- 提高手指或触控板拖拽效率
- 图片放大速率与画布放大速率一致

#### 前端改动点

- 调整缩放步进和拖拽阻尼
- 优化触控板和鼠标滚轮手感
- 统一节点缩放策略和画布缩放感知
- 图片节点和画布整体缩放视觉一致

#### 后端改动点

- 无核心后端改动

#### 数据库建议

- 无新增字段和表要求

### 5.2 视频结果比例与下载体验增强

#### 需求目标

- 视频节点比例与生成比例一致
- 可放大直接预览
- 在预览中一键下载

#### 前端改动点

- 结果按真实比例渲染
- 预览和下载入口固定在结果区
- 下载不依赖多层跳转

#### 后端改动点

- 统一返回下载地址和视频元数据

#### 数据库建议

- 优先复用 `task_results.meta`

## 6. 首页入口调整建议

首页建议简化为四个主入口：

- 工作台
- 画布
- 资源库
- 封装工作流

说明：

- 任务中心可以保留为工作台内的核心区域或次级导航
- 首页不再承担复杂统计展示职责
- 首页主要承担“进入生产路径”的导航职责

## 7. 推荐开发顺序

### 第一阶段

- P0 提示词输入稳定性修复
- P0 批量执行重构
- P0 视频节点交互重构
- P0 连线入口优化

### 第二阶段

- P0 资源库批量导入基础版
- P1 封装工作流

### 第三阶段

- P1 模特库（基于主体库扩展）
- P1 资源库体验重构
- P1 提示词 @ 图片资产

### 第四阶段

- P2 画布缩放优化
- P2 视频结果比例和下载体验增强

## 8. 当前版本结论

本批需求中，最重要的不是新增页面数量，而是三件事：

- 先把画布输入和批量执行修稳定
- 再把工作流封装做成复用能力
- 再把模特库作为主体库下的专业视图做深

其中，“模特库基于主体库扩展”是本阶段明确技术和产品原则，后续实现不再单独发展一套平行资源系统。

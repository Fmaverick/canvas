# AI 内容生产平台 PRD

## 1. 文档信息

- 文档名称：AI 内容生产平台产品需求文档
- 文档类型：PRD
- 当前版本：V1.0
- 文档目标：定义平台的业务目标、核心模块、系统约束、任务架构与 MVP 范围，作为产品、设计、前后端与 AI 基础设施协同的统一依据

## 2. 项目背景

当前业务需要同时调用多个图片模型、视频模型、音频模型与大语言模型完成内容生产，但不同供应商的接口协议、参数结构、任务状态和结果格式不一致，导致接入和维护成本高。

同时，平台侧只有一个 OpenAI Key，且该 Key 只能承载 50 并发，因此所有涉及 AI 的能力都必须经过统一调度和并发控制。尤其是视频模型大多采用异步任务模式，调用链路不是即时返回结果，而是需要：

- 先提交任务
- 持久化任务记录
- 周期性查询状态
- 在任务完成后批量回写结果
- 支持前端主动查询任务状态

除了 AI 调用层，业务本身还需要围绕主体素材、场景素材、指令资产和可视化画布形成完整创作闭环，使个人用户和团队用户都能在平台内管理资产并完成多步骤的 AI 生成流程。

## 3. 产品定位

本产品是一个面向个人创作者和团队内容生产场景的 AI 编排平台，整合以下能力：

- 团队空间与个人空间管理
- 主体库、场景库与指令库管理
- 可视化画布工作流
- 多模型统一接入与适配器层
- AI 任务队列、并发控制与异步轮询
- 平台级 Admin 后台

平台本质上不是单一的“调模型工具”，而是一个“素材资产管理 + 工作流画布 + AI 任务编排平台”。

## 4. 产品目标

### 4.1 业务目标

- 支持个人账号和团队空间两类协作模式
- 提供完整的主体素材、场景素材与预制 Prompt 管理能力
- 支持通过画布将文本、图片、视频、音频生成串联成完整工作流
- 统一接入多个图片模型、视频模型与音频模型，降低后续模型扩展成本
- 基于 OpenAI 提供统一 LLM 能力
- 在单 Key 的限制下实现稳定的并发控制与任务调度
- 为视频类异步模型提供标准队列、轮询和结果回写机制

### 4.2 成功指标

- 用户可以在一个画布中完成从文本生成到图片生成、音频生成再到视频生成的链路
- AI 请求全局并发不超过 50
- 视频任务具备可追踪、可轮询、可重试、可查询的标准生命周期
- 新增一个模型供应商时不需要改动业务层逻辑
- 团队用户可共享主体、场景、指令、画布和生成结果

## 5. 非目标

- 本期不做复杂计费、结算与套餐体系
- 本期不做企业级审批流程
- 本期不训练自有模型
- 本期不实现多人实时协同编辑同一画布
- 本期不做多 Key 调度系统，默认只有一个 OpenAI Key

## 6. 目标用户

### 6.1 个人创作者

- 需要快速生成文案、海报、短视频素材
- 希望有低门槛的素材管理与生成工作流

### 6.2 团队内容生产人员

- 需要共享主体信息、场景配置与指令模板
- 需要复用生成链路和结果
- 需要统一任务管理和空间权限

### 6.3 平台管理员

- 需要管理用户、空间、任务、模型开关和系统健康状态
- 需要处理失败任务、限流问题与异常供应商状态

## 7. 核心约束

- LLM 统一使用 OpenAI
- OpenAI 只有一个 Key，最大并发上限为 50
- 图片模型、视频模型与音频模型可能来自多个供应商，必须做统一抽象
- 视频任务为异步模式，必须支持任务查询与定时轮询
- Admin 后台密码不入库，使用环境变量读取
- 资源数据需要按空间隔离

## 10. 核心业务流程

### 10.1 用户登录与进入空间

1. 用户完成登录
2. 系统返回其可访问的空间列表
3. 用户选择进入个人空间或团队空间
4. 系统加载对应空间的资源和权限

### 10.2 素材管理流程

1. 用户创建或导入主体
2. 用户上传主体图片、描述和标签
3. 用户创建或导入场景
4. 用户上传场景图像、环境信息和标签
5. 用户保存常用指令模板

### 10.3 画布创作流程

1. 用户创建画布
2. 在画布中添加文本、图片、视频、音频节点
3. 节点下方填写生成内容输入
4. 选择主体、场景和指令作为上下文来源
5. 使用连接线将节点连接起来
6. 系统按照连接关系组织生成上下文
7. 用户触发某个节点生成

### 10.4 任务执行流程

1. 系统创建标准任务记录
2. 根据任务类型进入对应队列
3. 文本、图片或音频同步任务直接执行
4. 视频任务或异步音频任务提交后进入轮询流程
5. 系统更新任务状态并回写结果
6. 前端通过任务查询接口或主动轮询获取最新状态

## 12. 节点连接与上下文合并规则

### 12.1 设计目标

节点连接需要支持将上游节点输出与当前节点输入整合起来，形成新的生成任务，使用户可以搭建链式生成流程。

### 12.2 输入来源

- 当前节点手动输入
- 上游节点输出
- 产品库引用信息
- 模特库引用信息
- 系统模板
- 模型默认参数

### 12.3 合并优先级

- 用户显式输入
- 当前节点参数
- 上游节点输出
- 产品/模特引用上下文
- 系统模板默认值

### 12.4 不同节点类型的合并方式

#### 文本节点

- 将上游文本输出与当前输入拼接为新的 Prompt

#### 图片节点

- 上游文本作为图片 Prompt
- 上游图片作为参考图
- 产品和模特作为补充上下文

#### 视频节点

- 上游文本作为视频脚本或镜头说明
- 上游图片作为视觉参考或首帧参考
- 产品与模特信息作为视频内容约束条件

#### 音频节点

- 上游文本作为旁白、对白、歌词或音频脚本输入
- 上游视频可提取镜头节奏信息作为配音或配乐参考
- 上游图片可提供场景氛围描述，用于生成环境音或情绪音乐
- 产品信息可作为品牌口播与卖点内容来源

### 12.5 多上游节点规则

- 默认按连接顺序聚合
- 可支持后续增加优先级排序
- 若上游节点无有效输出，则跳过
- 若依赖节点失败，则当前节点禁止生成并提示用户处理

## 14. OpenAI 并发控制设计

### 14.1 设计背景

当前只有一个 OpenAI Key，并发上限为 50，因此平台必须在任务调度层做统一并发控制，避免超限导致任务失败。

### 14.2 调度策略

建议采用双层限流策略。

#### 第一层：全局并发池

- 维护全局信号量，最大值为 50
- 任意 OpenAI 请求发起前必须申请令牌
- 请求完成、失败或超时后释放令牌

#### 第二层：任务类型配额

建议默认按以下比例切分：

- 文本任务：20
- 图片相关提示优化任务：20
- 状态轮询及系统任务：10

该配额应支持在 Admin 后台动态调整。

### 14.3 调度原则

- 任何时刻全局活跃任务数不得超过 50
- 高成本任务优先入队，不直接抢占已有请求
- 单用户不允许独占所有并发
- 系统保留基础配额用于状态轮询和后台任务

## 15. 队列与异步任务系统

### 15.1 队列设计

建议最少拆分为以下队列：

- `text_generation_queue`
- `image_generation_queue`
- `audio_generation_queue`
- `video_submit_queue`
- `video_poll_queue`

### 15.2 文本任务流程

1. 创建任务记录
2. 加入文本生成队列
3. 获取并发令牌
4. 调用 OpenAI
5. 回写结果并释放令牌

### 15.3 图片任务流程

1. 创建任务记录
2. 加入图片生成队列
3. 调用图片模型适配器
4. 若同步返回，则直接完成
5. 若异步返回，则进入状态轮询

### 15.4 视频任务流程

1. 创建任务记录
2. 加入视频提交队列
3. 调用视频模型适配器提交任务
4. 返回 `provider_task_id`
5. 更新任务状态为 `dispatched` 或 `processing`
6. 写入轮询队列
7. 定时任务批量查询状态
8. 成功后回写视频结果与元数据

### 15.5 音频任务流程

1. 创建任务记录
2. 加入音频生成队列
3. 调用音频模型适配器
4. 若同步返回，则直接回写音频结果
5. 若异步返回，则记录 `provider_task_id` 并复用统一轮询机制

### 15.6 轮询机制

- 采用定时批处理任务
- 每 10 到 30 秒执行一次，周期可配置
- 每次选择 `next_poll_at <= now` 的任务
- 批量查询供应商状态
- 若仍在处理中，更新 `next_poll_at`
- 若成功，回写结果
- 若失败，记录失败原因
- 若超过最大轮询次数，标记为超时失败

### 15.7 任务状态查询接口

统一提供任务查询接口给前端和后台：

- 根据平台 `task_id` 查询任务状态
- 返回任务当前状态、进度、输出结果和错误信息
- 支持节点页面轮询刷新
- 支持 Admin 查看异常任务

### 15.8 重试机制

- 网络失败：自动重试 2 到 3 次
- 供应商限流：退避重试
- 参数错误：不自动重试
- 视频超时：支持手动重新提交
- 音频异步任务超时：支持手动重新提交

### 15.9 幂等策略

- 每次生成请求带 `request_id`
- 相同请求在短时间内重复触发时可识别为重复提交
- 避免重复创建供应商任务单

## 19. 异常与边界场景

- OpenAI 超限时，任务进入排队并提示系统繁忙
- 供应商返回提交成功但状态查询异常时，任务进入异常待重试状态
- 上游节点没有输出时，当前节点禁止执行
- 画布连接不允许形成环
- 删除被引用产品或模特时，系统提示先解除引用
- 用户关闭页面后，后台任务继续执行
- 任务结果资源失效时，支持重新获取或标记失效

## 20. 安全要求

- Admin 密码来自环境变量，不写入数据库
- 生产环境推荐存储哈希密码
- 任务结果访问必须校验空间权限
- 不向前端泄露供应商密钥
- 关键操作记录审计日志
- 控制失败重试次数，避免成本失控

## 21. 性能与稳定性要求

- 普通页面接口 P95 响应时间小于 500ms
- 任务创建接口 P95 响应时间小于 1s
- 视频状态同步到前端的目标延迟不超过 30s
- 全局并发不得超过 50
- 队列任务支持宕机恢复
- 关键任务状态必须持久化

## 25. 总体验收标准

- 用户可以在个人空间与团队空间中隔离管理资源
- Admin 可以通过环境变量密码登录后台
- 平台可以统一调用多个图片模型、视频模型和音频模型
- OpenAI 请求并发严格不超过 50
- 视频任务支持异步提交、轮询、查询和结果回写
- 音频任务支持同步或异步接入并可追踪状态
- 画布支持文本、图片、视频、音频节点
- 每个节点有独立输入框和生成配置区
- 节点支持连接并合并上游内容
- 用户可以在任务中心追踪任务生命周期
- 失败任务可以查看原因并重试

## 26. 产品结论

该平台的核心不是单个页面或单次调用，而是三层能力的稳定组合：

- 统一适配器层
- 全局并发控制层
- 异步任务与轮询层

这三层能力一旦打稳，前台无论是产品库、模特库还是画布节点，都可以持续扩展更多模型、更多节点类型和更多自动化内容生产场景。

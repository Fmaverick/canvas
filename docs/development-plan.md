## 3. 开发阶段规划

### Phase 0：开发准备

目标：

- 明确技术选型
- 补齐环境配置
- 确认第一批模型供应商

产出物：

- 数据库与 Redis 可用
- 对象存储可用
- 环境变量模板
- 模型名单与能力清单

### Phase 1：基础能力

目标：

- 落地用户体系与空间体系
- 完成基础鉴权
- 完成 Admin 登录能力

开发项：

- 用户表与空间表
- 空间成员关系
- 用户登录注册
- 空间切换
- Admin 环境变量鉴权

完成标志：

- 用户可登录
- 用户可创建或进入空间
- Admin 可登录后台

### Phase 2：资产管理

目标：

- 落地主体库、场景库与指令库
- 完成上传与资源管理

开发项：

- 主体 CRUD
- 场景 CRUD
- 指令 CRUD
- 资产上传
- 标签检索
- 资产引用关系

完成标志：

- 主体、场景与指令可被创建、编辑、删除、筛选
- 资产可上传并持久化

### Phase 3：画布基础

目标：

- 完成画布、节点、连接管理
- 支持文本 / 图片 / 视频 / 音频节点
- 支持节点复制与节点模板

开发项：

- 画布 CRUD
- 节点 CRUD
- 节点复制
- 边 CRUD
- 防环校验
- 节点模板 CRUD
- 从模板创建节点
- 节点配置面板数据结构

完成标志：

- 用户可在画布中拖拽并保存节点
- 节点之间可连接
- 用户可复制节点并复用节点模板

### Phase 4：AI 网关与任务系统

目标：

- 落地统一适配器接口
- 接通任务表与队列
- 实现并发控制

开发项：

- 统一 AI 请求模型
- 适配器注册中心
- OpenAI 文本适配器
- 图片适配器
- 视频适配器
- 音频适配器
- 全局并发控制
- 任务入队与消费

完成标志：

- 文本、图片、视频、音频任务都可创建平台任务
- OpenAI 并发不超过 50

### Phase 5：异步轮询与任务中心

目标：

- 完成异步媒体轮询
- 完成任务详情追踪

开发项：

- 视频任务轮询
- 异步音频任务轮询
- 状态查询接口
- 失败重试
- 节点状态回填

完成标志：

- 用户可在任务中心看到完整生命周期
- 媒体任务完成后能自动回写结果

### Phase 6：Admin 与监控

目标：

- 建立平台级管理与监控能力

开发项：

- Admin 仪表盘
- 模型开关管理
- 队列监控
- 审计日志
- 失败任务排查

完成标志：

- Admin 可查看运行数据和异常任务

## 8. 风险与应对

### 风险 1：OpenAI 并发超限

应对：

- 统一 semaphore
- 请求前置排队
- 后台配额调节

### 风险 2：视频或音频供应商状态不稳定

应对：

- 统一重试
- 轮询兜底
- 错误码归一

### 风险 3：画布节点逻辑复杂

应对：

- 节点与任务解耦
- 先做单节点执行
- 再做链式执行优化

### 风险 4：开发范围失控

应对：

- 严格按 Phase 推进
- MVP 只接首批模型
- 复杂能力延后
